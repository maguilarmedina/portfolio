Primeros pasos sobre Python

Programación

4 de enero 2025

Primero de todo, tenemos que elegir un editor de texto plano. En mi caso, yo uso Visual Studio Code. Python es el lenguaje de programación más fácil para empezar, así que aquí dejo lo más básico de él.
La extensión del archivo python es .py, y es necesario descargar el lenguaje, desde su página oficial. Una vez instalado, ya podremos empezar a programar. Lo más común es empezar creando un archivo que muestre por la pantalla "Hola mundo". Para hacer esto en python, simplemente es con la línea print("Hola mundo") . Ahora, ejecutamos el archivo python (sin tener que compilarlo) o desde el propio Visual Studio, arriba a la derecha, donde sale para ejecutarlo.
print("") sirve para mostrar texto por pantalla. Las variables sirven para almacenar un texto, número, etc. También podemos usarlo para preguntarle al usuario algo y que lo almacene, ya sea para mostrarlo por la pantalla, sumarlo a algo, etc. En este caso, podríamos hacerlo con variable1=input("Introduce la variable: ") ; tendremos que introducir un valor y se guardará en la variable variable1.

Hay varios tipos de datos: numéricos (int), texto (string), bool (verdadero/falso), float para números con decimales. Para sumar, restar, multiplicar y dividir números se hace con +, -, * y /.

Normalización de entidades

Base de datos

22 de noviembre 2024

Estos últimos días en clase hemos estado viendo la normalización de las tablas de base de datos.
La normalización consiste en la detección y corección de problemas que suelen surgir de una mala realización del modelo entidad relación, aunque también pueden salir en tablas bien hechas; por eso se recomienda revisar las tablas después de terminarlas.
Para normalizar, primero debemos identificar si una entidad es dependiente funcional de otra entidad con la pregunta "un x concreto tiene uno o varios y" (por ejemplo, un gato específico tiene un solo nombre y no varios. Este en caso gato-específico sí es determinante funcional de un nombre (pero no al revés, pues un nombre lo pueden tener varios gatos.)). Aplicamos esta pregunta a todos los elementos de la tabla, luego hacemos lo mismo pero por parejas y por último agrupamos de más en más.
Una vez obtenidas todas las dependencias funcionales, analizamos cuales cumplen los criterios FNBC, siendo estos los siguientes:
-Las D.F que lo sean de todos deben ser claves candidatas,y deben estar en las PK o en su defecto ser UNN. Si pasara al revés, (es decir, una entidad que es PK o UNN cuando no debería serlo, le quitamos PK o UNN)
-Si una entidad no es D.F de todos sino de uno o varios, hacemos una tabla aparte en la que ella es la PK y abajo las otras entidades. Esta estará enlazada a la tabla original con una FK.
